# FashionClaw 未来开发路线图

本文档列出 FashionClaw 从演示原型到生产系统所需的 Agent 接入点和外部 API 集成需求。

---

## 第一阶段：核心功能增强

### 1. 智能监测 Agent (Monitor Agent)

**现状：** 仅基于简单天数阈值判断

**未来 Agent 能力：**
- [ ] **穿着频率预测 Agent**
  - 分析用户历史穿着模式（季节、场合、搭配习惯）
  - 预测未来 90 天穿着概率
  - 接入用户日历数据识别特殊场合需求

- [ ] **情感价值评估 Agent**
  - 识别具有纪念意义的衣物（礼物、纪念品等）
  - 通过用户行为（查看次数、收藏等）判断情感价值
  - 建议保留而非出售

**所需 API：**
| API | 用途 | 服务商参考 |
|-----|------|-----------|
| 用户日历 API | 获取 upcoming events，判断是否需要特定服装 | Google Calendar, Apple Calendar |
| 天气 API | 结合季节变化预测穿着需求 | OpenWeatherMap, 彩云天气 |
| 用户行为追踪 API | 记录用户在 App 内的浏览、收藏行为 | 自建或 Mixpanel/Amplitude |

---

### 2. 价格评估 Agent (Pricing Agent)

**现状：** 随机生成市场价格

**未来 Agent 能力：**
- [ ] **实时市场定价 Agent**
  - 抓取闲鱼、转转等平台同款商品实时售价
  - 分析品牌、成色、尺码对价格的影响
  - 预测最佳出售时机（季节因素）

- [ ] **动态定价策略 Agent**
  - 根据商品热度调整定价
  - 自动降价促销策略（7天/14天/30天阶梯降价）

**所需 API：**
| API | 用途 | 服务商参考 |
|-----|------|-----------|
| 闲鱼/转转爬虫 API | 获取同款商品历史成交价格 | 自建爬虫或第三方数据服务 |
| 电商价格比较 API | 新品价格参考，计算折旧率 | 慢慢买、什么值得买 |
| 品牌数据库 API | 识别品牌档次、保值率 | 自建品牌库或对接得物 |

---

### 3. 智能匹配 Agent (Buyer Matching Agent)

**现状：** 随机生成买家和出价

**未来 Agent 能力：**
- [ ] **真实买家匹配 Agent**
  - 对接平台真实求购需求
  - 基于地理位置匹配最近买家（降低物流成本）
  - 买家偏好匹配（风格、尺码、品牌偏好）

- [ ] **议价 Agent**
  - 自动与买家进行价格谈判
  - 设置最低接受价格和谈判策略
  - 多买家竞价模式

**所需 API：**
| API | 用途 | 服务商参考 |
|-----|------|-----------|
| 闲鱼开放平台 API | 发布商品、获取求购信息、消息通知 | 闲鱼开放平台（需申请） |
| 转转开放平台 API | 多平台发布，扩大曝光 | 转转开放平台 |
| 即时通讯 API | 自动回复买家咨询、议价 | 环信、网易云信 |

---

### 4. 信用评估 Agent (Credit Assessment Agent)

**现状：** 返回固定的 "Excellent" 评级

**未来 Agent 能力：**
- [ ] **平台信用聚合 Agent**
  - 对接多个平台信用体系
  - 分析买家历史交易数据（退货率、纠纷率、付款及时性）
  - 识别高风险买家（职业差评师、到手刀等）

- [ ] **社交信用交叉验证 Agent**
  - 关联买家社交平台行为
  - 识别虚假账号和诈骗风险

**所需 API：**
| API | 用途 | 服务商参考 |
|-----|------|-----------|
| 闲鱼用户信用 API | 获取芝麻信用分、闲鱼信用等级 | 闲鱼开放平台 |
| 转转用户信用 API | 获取转转信用分 | 转转开放平台 |
| 支付宝芝麻信用 API | 深度信用评估 | 蚂蚁金服开放平台 |
| 反欺诈 API | 识别黑名单用户、设备指纹 | 同盾、数美科技 |

--- 

## 第二阶段：用户体验增强

### 5. 图像识别 Agent (Vision Agent)

**新增 Agent：**
- [ ] **自动分类 Agent**
  - 用户上传衣物照片，自动识别类别（上衣/裤子/裙子）
  - 识别品牌、款式、颜色
  - 自动填写商品信息

- [ ] **成色评估 Agent**
  - 通过照片识别磨损程度
  - 自动生成商品描述（"9成新，无明显瑕疵"）

- [ ] **穿搭推荐 Agent**
  - 基于用户衣橱生成搭配建议
  - 提高衣物利用率，减少闲置

**所需 API：**
| API | 用途 | 服务商参考 |
|-----|------|-----------|
| 图像分类 API | 衣物类型识别 | 阿里云视觉智能、腾讯云AI |
| OCR API | 识别品牌标签、水洗标 | 百度OCR、腾讯OCR |
| 颜色识别 API | 主色调、配色方案提取 | 自建或 Color API |
| 穿搭推荐 API | 基于深度学习的搭配推荐 | 自建模型或对接穿搭App |

---

### 6. 个性化推荐 Agent (Personalization Agent)

**新增 Agent：**
- [ ] **购物决策 Agent**
  - 分析用户衣橱，识别缺失单品
  - 推荐新购买时考虑与现有衣物搭配性

- [ ] **断舍离建议 Agent**
  - 定期生成衣橱分析报告
  - 建议哪些衣物该出售、捐赠、改造

**所需 API：**
| API | 用途 | 服务商参考 |
|-----|------|-----------|
| 电商推荐 API | 获取流行趋势、新品推荐 | 淘宝开放平台、京东开普勒 |
| 时尚趋势 API | 获取当季流行款式、颜色 | WGSN、POP时尚 |

---

## 第三阶段：商业拓展

### 7. 多渠道分发 Agent (Multi-channel Agent)

**新增 Agent：**
- [ ] **一键多发 Agent**
  - 一次发布，同步到闲鱼、转转、95分等多平台
  - 自动调整各平台商品描述格式
  - 统一管理多平台询盘

- [ ] **海外销售 Agent**
  - 识别适合海外市场的商品（奢侈品、 vintage）
  - 自动发布到 Depop、Poshmark、Vestiaire Collective

**所需 API：**
| API | 用途 | 服务商参考 |
|-----|------|-----------|
| 闲鱼/转转/95分 API | 国内二手平台 | 各平台开放平台 |
| Depop API | 海外年轻用户市场 | Depop Developer |
| Poshmark API | 北美二手时尚 | Poshmark API |

---

### 8. 支付与结算 Agent (Payment Agent)

**新增 Agent：**
- [ ] **资金托管 Agent**
  - 交易资金第三方托管，确认收货后自动结算
  - 支持分期付款、定金模式

- [ ] **收益分析 Agent**
  - 统计用户出售收益
  - 生成月度/年度衣橱财务报告

**所需 API：**
| API | 用途 | 服务商参考 |
|-----|------|-----------|
| 支付宝 API | 收款、转账、资金托管 | 蚂蚁金服开放平台 |
| 微信支付 API | 微信生态收款 | 微信支付商户平台 |
| 银行存管 API | 资金托管合规 | 各银行开放银行服务 |

|-----|-----------|---------|

## 技术架构演进建议

### 当前（演示版）
```
Gradio UI → LangGraph → Mock APIs → JSON File
```

### 短期目标（MVP）
```
Mobile App → LangGraph → 闲鱼/支付 API → SQLite/PostgreSQL
                    ↓
              Redis Cache
```

### 长期目标（生产系统）
```
Multi-client → API Gateway → Agent Orchestrator → External APIs
(App/Web/IoT)              (LangGraph/LlamaIndex)
                                   ↓
                    ┌──────────────┼──────────────┐
                    ↓              ↓              ↓
              Agent 1         Agent 2         Agent N
           (Monitor)       (Pricing)       (Payment)
                    ↓              ↓              ↓
              Vector DB      Time-series    Analytics
              (Pinecone)     (InfluxDB)     (ClickHouse)
```

## 实施建议

### 第一阶段（1-2个月）
1. 申请闲鱼开放平台权限（流程较长，需提前开始）
2. 接入芝麻信用基础版
3. 实现基础定价模型

### 第二阶段（2-3个月）
4. 接入图像识别 API，实现拍照自动上架
5. 开发价格监控爬虫，积累数据训练定价模型
6. 接入智能客服降低人工成本

### 第三阶段（3-6个月）
7. 开发穿搭推荐模型
8. 拓展海外平台渠道
9. 接入数据分析平台，生成用户衣橱洞察报告
